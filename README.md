# Лабораторна робота 3: Об’єктно-орієнтоване програмування у JavaScript

## Мета
- Закріпити базові прийоми створення та використання об’єктів у JavaScript без синтаксису класів ES6.
- Освоїти механізм прототипного наслідування та написання конструкторів.
- Використати синтаксис класів ES6 для побудови ієрархій геометричних фігур.
- Попрактикуватися у створенні фабричних функцій та замикань.

## Структура проєкту
- `oop-lab/index.html` – стартова сторінка лабораторної роботи.
- `oop-lab/script.js` – реалізація всіх пунктів завдання (1.2.3–1.2.31).

2. Запустіть консоль розробника (F12 або Ctrl+Shift+I) та перейдіть на вкладку Console.

## Виконані етапи
- Створені об’єкти `car1` та `car2` двома різними способами та додано методи `drive`.
- Реалізовано конструктор `Truck`, метод `trip` та метод `AssignDriver` через прототип, продемонстровано роботу на двох екземплярах.

- Створено фабричну функцію `Triangular`, функцію-вищий порядок `PiMultiplier` та замикання `Painter` із демонстрацією роботи.

## Контрольні запитання (короткі відповіді)
1. **Як реалізовується успадкування у JS?** Через прототипи (ланцюжок `[[Prototype]]`) або через синтаксис `class`/`extends`, який все одно використовує прототипи під капотом.
2. **Які способи реалізації ООП є у JS?** Прототипно-орієнтоване (через об’єкти та `Object.create`), функціонально-конструкторне (через функції-конструктори + `new`) та класове (ES6 `class`).
3. **Які способи реалізації успадкування у JS Ви знаєте?** Встановлення прототипу вручну (`Object.create`, `Object.setPrototypeOf`), використання функцій-конструкторів з прототипами, синтаксис `class`/`extends`.
4. **Як задати ґеттер у JS?** Використати ключове слово `get` у літералі об’єкта або класі, наприклад `get value() { return this._value; }`.
5. **Як задати сеттер у JS?** Аналогічно з ключовим словом `set`: `set value(newValue) { this._value = newValue; }`.
6. **Що таке factory function у JS?** Це функція, яка повертає новий об’єкт без використання ключового слова `new`, зазвичай інкапсулюючи логіку створення.
7. **Які особливості написання конструкторів?** Конструктор викликається через `new`, ініціалізує властивості екземпляра через `this`, може додавати методи, але краще винести спільні методи в прототип.

Виконано всі пункти лабораторної роботи: від базових об’єктів і функцій-конструкторів до побудови класової ієрархії, роботи з замиканнями та фабричними функціями. Отримані практичні навички демонструють розуміння різних парадигм ООП у JavaScript та вміння комбінувати їх у межах одного проєкту.
